# ACSD-68490: botão **[!UICONTROL Add New Attribute]** visível para o administrador restrito durante a criação do produto configurável

O patch ACSD-68490 corrige o problema em que o botão **[!UICONTROL Add New Attribute]** está visível para um administrador restrito durante a criação do produto configurável. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 está instalado. A ID do patch é ACSD-68490. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6 - 2.4.6-p13

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

O botão **[!UICONTROL Add New Attribute]** está visível para um administrador restrito durante a criação do produto configurável.

<u>Etapas a serem reproduzidas</u>:

1. Crie um site adicional, loja e visualização de loja.
1. Crie uma função de administrador que tenha acesso somente ao novo site e atribua todos os recursos.
1. Crie um usuário administrador restrito e atribua a nova função a esse usuário.
1. Faça logon no Admin como o usuário restrito.
1. Crie um novo produto configurável.


<u>Resultados esperados</u>:

O botão **[!UICONTROL Create New Attribute]** não está disponível para o usuário administrador restrito.

<u>Resultados reais</u>:

Uma página 404 é exibida.

## Aplicar o patch

Para aplicar patches individuais, use os links a seguir, dependendo do método de implantação:

* Adobe Commerce ou Magento Open Source local: [[!DNL Quality Patches Tool] > Uso](/help/tools/quality-patches-tool/usage.md) no guia [!DNL Quality Patches Tool].
* Adobe Commerce na infraestrutura em nuvem: [Atualizações e patches > Aplicar patches](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/upgrade/apply-patches.html?lang=pt-BR) no guia do Commerce na infraestrutura em nuvem.
