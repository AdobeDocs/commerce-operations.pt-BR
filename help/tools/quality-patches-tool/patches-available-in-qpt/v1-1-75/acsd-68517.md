# ACSD-68517: corrige um erro de reenvio de formulário nas páginas **[!UICONTROL Catalog]** e **[!UICONTROL Catalog Search]**

O patch ACSD-68517 corrige o problema em que um erro de reenvio de formulário é exibido ao usar a seta para trás do navegador nas páginas **[!UICONTROL Catalog]** e **[!UICONTROL Catalog Search]**. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 está instalado. A ID do patch é ACSD-68517. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6-p12

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.6-p13

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

As páginas **[!UICONTROL Catalog]** e **[!UICONTROL Catalog Search]** exibem uma caixa de diálogo **[!UICONTROL Confirm Form Resubmission]** ao usar a seta para trás do navegador após alterar a ordem da lista de produtos, devido ao tratamento inadequado de solicitações POST em cache.

<u>Etapas a serem reproduzidas</u>:

1. Defina a configuração de **[!UICONTROL Remember Category Pagination]** como *Sim*.
2. Procure um produto na loja.
3. Altere a ordem da lista para qualquer coisa diferente do padrão ****[!UICONTROL Relevance]** na página de resultados da pesquisa.
4. Ir para uma página de produto.
5. Clique na seta para trás do navegador.
 

<u>Resultados esperados</u>:

A página de resultados da pesquisa é carregada normalmente ao clicar na seta para trás do navegador.

<u>Resultados reais</u>:

A caixa de diálogo **[!UICONTROL Confirm Form Resubmission]** é exibida ao clicar na seta para trás do navegador.
