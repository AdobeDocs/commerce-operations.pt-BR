# ACSD-68793: Produtos válidos são rejeitados incorretamente durante a atribuição do catálogo compartilhado

O patch ACSD-68793 corrige o problema em que produtos válidos são rejeitados incorretamente durante a atribuição do catálogo compartilhado. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 está instalado. A ID do patch é ACSD-68793. Observe que esse problema está agendado para ser corrigido no Adobe Commerce [!DNL B2B] 1.5.3.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) [!DNL B2B] 1.5.2-p1

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) [!DNL B2B] 1.5.1 - 1.5.2-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Os produtos válidos são rejeitados incorretamente durante a atribuição de catálogo compartilhado por meio da API [!DNL REST], e o sistema retorna um erro informando que as SKUs existentes não existem.

<u>Etapas a serem reproduzidas</u>:

1. Crie vários produtos.
1. Na barra lateral **[!UICONTROL Admin]**, navegue até **[!UICONTROL Catalog]** > **[!UICONTROL Shared Catalog]**.
1. Clique em **[!UICONTROL Add Shared Catalog]** e crie um catálogo compartilhado.
1. Atribua todos os produtos ao catálogo compartilhado criado.
1. Envie uma solicitação de API [!DNL REST] para `/rest/V1/sharedCatalog/3/assignProducts` com a lista de SKUs de produtos existentes para atribuí-los ao catálogo compartilhado.

<u>Resultados esperados</u>:

Uma resposta bem-sucedida é retornada. Os erros devem ser retornados somente para produtos que não existem.

<u>Resultados reais</u>:

O sistema retorna o erro *O produto solicitado não existe: %sku.* para um SKU existente.
