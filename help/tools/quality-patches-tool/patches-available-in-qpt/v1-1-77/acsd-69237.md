# ACSD-69237: `sales_*_async_insert` os trabalhos cron processam somente 100 entradas por execução

O patch ACSD-69237 corrige o problema em que os trabalhos do cron `sales_*_async_insert` processavam apenas 100 entradas por execução ao inserir registros nas tabelas de grade de vendas, em vez de processar todas as linhas pendentes em lotes. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 está instalado. A ID do patch é ACSD-69237. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.5-p15

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Os trabalhos do cron `sales_*_async_insert` processam apenas 100 entradas por execução ao inserir registros nas tabelas da grade de vendas, mesmo quando existem mais linhas pendentes, resultando em atualizações incompletas em tabelas como `sales_order_grid`.

<u>Etapas a serem reproduzidas</u>:

1. Gerar 800 pedidos no sistema.
1. Habilite a indexação assíncrona para as grades navegando até: **[!UICONTROL Configuration]** > **[!UICONTROL Advanced]** > **[!UICONTROL Developer]** > **[!UICONTROL Grid Settings]**.
1. Verifique a contagem de pedidos no banco de dados:

   ```
   SELECT COUNT(*) FROM sales_order_grid;
   ```

   O resultado deve ser 800.
1. Exclua o conteúdo da tabela `sales_order_grid`:

   ```
   DELETE FROM sales_order_grid;
   ```

1. Execute os trabalhos cron que processam as filas `sales_*_async_insert`.
1. Verifique a contagem de pedidos no banco de dados novamente:

   ```
   SELECT COUNT(*) FROM sales_order_grid;
   ```

<u>Resultados esperados</u>:

Todas as 800 ordens ausentes são inseridas na tabela `sales_order_grid` por meio de uma única execução de trabalho cron `sales_grid_order_async_insert`.

<u>Resultados reais</u>:

Apenas 100 das 800 ordens ausentes são inseridas na tabela `sales_order_grid` durante uma única execução do trabalho cron `sales_grid_order_async_insert`.
