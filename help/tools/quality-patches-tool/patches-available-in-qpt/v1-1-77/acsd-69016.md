# ACSD-69016: Preço especial não é aplicado em sites com fusos horários diferentes

O patch ACSD-69016 corrige o problema em que as solicitações de API da atualização de produto [!DNL REST] falham se não houver um armazenamento com o código padrão, o que impede que preços especiais entrem em vigor para sites criados em fusos horários diferentes. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 está instalado. A ID do patch é ACSD-69016. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p1

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

As solicitações de API [!DNL REST] de atualização de produto falharão se não houver loja com o código padrão, o que impede que preços especiais entrem em vigor para sites criados em fusos horários diferentes.

<u>Etapas a serem reproduzidas</u>:

1. No painel **[!UICONTROL Admin]**, vá para **[!UICONTROL Stores]** >**[!UICONTROL  All Stores]** > **[!UICONTROL Default Store View]**.
1. Alterar o valor do código da exibição de loja padrão.
1. Envie uma solicitação de API [!DNL REST] para atualizar um produto.

<u>Resultados esperados</u>:

A solicitação é concluída com sucesso e o produto é atualizado.

<u>Resultados reais</u>:

A solicitação falha e o produto não é atualizado.

## Aplicar o patch

Para aplicar patches individuais, use os links a seguir, dependendo do método de implantação:
