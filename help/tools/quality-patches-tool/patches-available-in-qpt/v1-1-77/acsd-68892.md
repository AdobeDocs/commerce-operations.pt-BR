# ACSD-68892: comportamento de cache [!DNL Fastly] inconsistente para páginas que podem ser armazenadas em cache

O patch ACSD-68892 corrige o problema em que páginas armazenáveis em cache não são armazenadas ou fornecidas corretamente do cache do [!DNL Fastly], resultando em comportamento de cache inconsistente e desempenho reduzido. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 está instalado. A ID do patch é ACSD-68892. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p2

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

As páginas que podem ser armazenadas em cache não são armazenadas ou veiculadas corretamente no cache do [!DNL Fastly], resultando em um comportamento inconsistente de armazenamento em cache e em desempenho reduzido.

<u>Etapas a serem reproduzidas</u>:

1. Habilite o cache do [!DNL Fastly].
1. Crie uma nova visualização de site/loja/loja.
1. Adicionar o código do site secundário a `index.php`:

```
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. Verificar cabeçalho de controle de cache usando curl:

```
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u>Resultados esperados</u>:

Páginas armazenáveis em cache são armazenadas em cache em [!DNL Fastly].

<u>Resultados reais</u>:

Páginas armazenáveis em cache não são armazenadas em cache em [!DNL Fastly], a `max-age=0` retornou no cabeçalho.

```
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```
