# ACSD-69494: solicitações de reembolso assíncronas com is_online não acionam reembolsos online

O patch ACSD-69494 corrige o problema com as operações de reembolso assíncronas em que as solicitações de reembolso com o parâmetro `is_online` não são processadas corretamente. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 está instalado. A ID do patch é ACSD-69494. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

As solicitações de reembolso assíncronas com o parâmetro `is_online` não são processadas corretamente.

<u>Etapas a serem reproduzidas</u>:

1. Configurar PayPal.
1. Configurar [!DNL RabbitMQ].
1. Crie um produto.
1. Check-out com PayPal.
1. Criar uma fatura.
1. Crie um memorando de crédito usando a API [!DNL REST] assíncrona.
1. Execute a fila assíncrona.
1. Faça logon no gateway de pagamento e verifique o status da transação.

<u>Resultados esperados</u>:

A transação é mostrada como **Reembolsada** quando um reembolso online é executado.

<u>Resultados reais</u>:

A transação é exibida como **Concluída**.
