# ACP2E-4540: A biblioteca Fotorama não é carregada corretamente, portanto, somente a primeira imagem anexada é visível

O patch ACP2E-4540 corrige o problema em que a biblioteca Fotorama não é carregada corretamente, fazendo com que apenas a primeira imagem anexada seja exibida. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4540. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p7

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A biblioteca Fotorama não é carregada corretamente, o que faz com que somente a primeira imagem anexada fique visível.

<u>Etapas a serem reproduzidas</u>:

1. Crie um produto simples e anexe várias imagens a ele.
1. Abra o produto na loja.

<u>Resultados esperados</u>:

A galeria de imagens exibe todas as imagens anexadas.

<u>Resultados reais</u>:

1. Somente a primeira imagem é visível.
1. O console do navegador mostra o seguinte erro:

```
Javascript error(severe) - http://magento.com/static/version1717863530/frontend/magento/luma/en_us/mage/gallery/gallery.js 309:29 uncaught type error: $fotorama element.fotorama is not a function
```

## Aplicar o patch

Para aplicar patches individuais, use os links a seguir, dependendo do método de implantação:

* Adobe Commerce ou Magento Open Source local: [[!DNL Quality Patches Tool] > Uso](/help/tools/quality-patches-tool/usage.md) no guia [!DNL Quality Patches Tool].
* Adobe Commerce na infraestrutura em nuvem: [Atualizações e patches > Aplicar patches](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/upgrade/apply-patches.html?lang=pt-BR) no guia do Commerce na infraestrutura em nuvem.
