# ACP2E-4613: O diretório de mídia grande causa atrasos no carregamento da árvore de diretório da Galeria de Mídia

O patch ACP2E-4613 corrige o problema em que grandes estruturas de diretórios de mídia causam respostas lentas de `gettree`, levando a tempos de carregamento estendidos para a árvore de diretórios **[!UICONTROL Media Gallery]**. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4613. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Estruturas de diretório de mídia grandes causam respostas lentas de `gettree`, levando a tempos de carregamento estendidos para a árvore de diretórios [!UICONTROL Media Gallery].

<u>Pré-requisitos</u>:

1. Existe um grande número de produtos com várias imagens.

<u>Etapas a serem reproduzidas</u>:

1. Gerar miniaturas de imagem para aumentar o espaço da mídia:

   ```
   bin/magento catalog:images:resize
   ```

1. Crie vários subdiretórios em `pub/media/catalog/category` (Por exemplo: test, test2, test3, test4 e pastas aninhadas adicionais, conforme necessário).
1. Copie um grande conjunto de arquivos de mídia de catálogo nos diretórios recém-criados para aumentar significativamente o tamanho da árvore de diretórios de mídia, por exemplo:

   ```
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test/
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test2/
   (...)
   ```

1. Faça logon no painel Admin e navegue até **[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**.
1. Observe que a solicitação `admin/media_gallery/directories/gettree/?isAjax=true` leva muito tempo para ser concluída.

<u>Resultados esperados</u>:

A árvore de diretórios é carregada em alguns segundos.

<u>Resultados reais</u>:

A árvore de diretórios leva vários minutos para carregar, dependendo do número de subdiretórios em `pub/media`.
