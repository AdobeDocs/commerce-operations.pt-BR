# ACP2E-4448: As alterações de configuração feitas durante as interrupções de [!DNL Redis] não refletem após [!DNL Redis] ser recuperado, e os valores obsoletos persistem

O patch do ACP2E-4448 corrige o problema em que o cache local e o cache Redis remoto podem ficar dessincronizados quando o Redis está indisponível, e as alterações de configuração feitas durante as interrupções do Redis não refletem após a recuperação do Redis, fazendo com que os valores obsoletos persistam. A correção garante a sincronização correta em todos os caches e preserva os valores salvos mais recentes mesmo quando o Redis está inativo. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4448. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p1

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

O cache L2, o cache local, o cache obsoleto e o Redis ficaram não sincronizados quando o Redis estava indisponível. As alterações de configuração feitas durante as interrupções do Redis não foram refletidas após a recuperação do Redis, fazendo com que os valores obsoletos persistam.

<u>Etapas a serem reproduzidas</u>:

1. Configure o cache L2 usando o back-end do `RemoteSynchronizedCache` com Redis como back-end remoto e o cache de arquivos como back-end local.
1. Navegue até **Configuração** > **Geral** > **Contato** > **Opções de Email** > **Enviar Emails para**.
1. Atualize o valor **Enviar Emails para** para _100@example.com_.
1. Atualize a página várias vezes para confirmar que o valor está armazenado em cache.
1. Pare o serviço Redis para simular uma interrupção de serviço.
1. Atualize o valor **Enviar Emails para** para _200@example.com_.
1. Inicie o serviço Redis.
1. Atualize a página de configuração.

<u>Resultados esperados</u>:

Deve mostrar o novo valor de configuração _200@example.com_.

<u>Resultados reais</u>:

Ele mostra o antigo valor de configuração _100@example.com_.
