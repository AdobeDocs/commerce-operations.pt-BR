# ACP2E-4507: a configuração [!UICONTROL Password Options] não se aplica às solicitações de redefinição de senha do cliente feitas por meio de mutações do GraphQL

O patch ACP2E-4507 corrige o problema em que a configuração **[!UICONTROL Password Options]** não é aplicada para solicitações de redefinição de senha do cliente feitas por meio de mutações do GraphQL. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4507. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A configuração **[!UICONTROL Password Options]** não é aplicada para solicitações de redefinição de senha do cliente feitas por meio de mutações do GraphQL.

<u>Etapas a serem reproduzidas</u>:

1. Verifique e confirme as opções de senha do cliente no
   **[!UICONTROL Store]** > **[!UICONTROL Configurations]** > **[!UICONTROL Customers]** > **[!UICONTROL Customer Configuration]** > **[!UICONTROL Password Options]**.
1. Envie um email de redefinição de senha usando a mutação do GraphQL `requestPasswordResetEmail`.
1. Redefina a senha usando a mutação do GraphQL `resetPassword` usando o token do email enviado.
1. Repita as mesmas etapas para redefinir a senha várias vezes.

<u>Resultados esperados</u>:

As redefinições de senha do cliente acionadas pelo GraphQL se comportam da mesma forma que as redefinições de senha baseadas na Web e respeitam as definições de configuração do [!UICONTROL Admin].

<u>Resultados reais</u>:

As solicitações de redefinição de senha feitas por meio do GraphQL podem ser acionadas um número ilimitado de vezes e não impõem os limites definidos na configuração de senha do cliente [!UICONTROL Admin].
