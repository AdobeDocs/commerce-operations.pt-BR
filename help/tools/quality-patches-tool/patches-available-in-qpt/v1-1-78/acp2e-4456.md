# ACP2E-4456: Cancelar um pedido com uma mutação do GraphQL não move pedidos pagos inteiramente com cartões-presente para o status Fechado

O patch do ACP2E-4456 corrige o problema em que o cancelamento de pedido do GraphQL não faz a transição de pedidos pagos com cartão-presente para o status [!UICONTROL Closed]. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4456. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Ocorre um problema em que o cancelamento de um pedido por meio de uma mutação do GraphQL não faz a transição de pedidos pagos com cartão-presente para o status [!UICONTROL Closed].

<u>Etapas a serem reproduzidas</u>:

1. Crie um cartão-presente de qualquer tipo.
1. Crie um produto simples.
1. Habilitar **[!UICONTROL Free Shipping]**.
1. Habilite o cancelamento do pedido GraphQL navegando até **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL Sales]** > **[!UICONTROL Order cancellation]** e defina **[!UICONTROL Order cancellation through GraphQL]** como *Sim*.
1. Faça logon na conta da loja.
1. Compre um vale-presente, gere uma fatura para o pedido e anote o número do vale-presente.
1. Adicione o produto simples ao carrinho.
1. Selecione **[!UICONTROL Free Shipping]** durante o check-out.
1. Conclua o pagamento usando o cartão-presente.
1. Faturar a ordem.
1. Cancele a ordem usando a seguinte mutação do GraphQL:

   ```
   mutation {
   	cancelOrder(input: { order_id: "Nw==", reason: "Other" }) {
   		error
   		order {
   			carrier
   			created_at
   			email
   			gift_receipt_included
   			grand_total
   			id
   			increment_id
   			number
   			order_date
   			shipping_method
   			status
   		}
   	}
   }
   ```

<u>Resultados esperados</u>:

O status do pedido muda para Cancelado.

<u>Resultados reais</u>:

O pedido não é cancelado.
