# ACP2E-4555: Os números de telefone que contêm pontos ou barras não são validados corretamente

O patch ACP2E-4555 corrige o problema em que números de telefone modernos contendo `.` ou `/` não são validados. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4555. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Números de telefone modernos contendo `.` ou `/` não são validados.

<u>Etapas a serem reproduzidas</u>:

1. Faça logon como cliente na loja.
1. Adicione um novo endereço ao catálogo de endereços com um número de telefone, por exemplo: `06.76.40.32.22` ou `+43680/2149568`.
1. Salve os detalhes do endereço.

<u>Resultados esperados</u>:

É possível usar formatos de telefone modernos com pontos e barras.

<u>Resultados reais</u>:

Falha na validação do campo de telefone devido aos `.` ou `/` caracteres.
