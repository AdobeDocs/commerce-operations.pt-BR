# ACP2E-4609: A página **[!UICONTROL My Quotes]** não mostra cotações quando algumas cotações contêm produtos excluídos

O patch ACP2E-4609 corrige o problema em que a página **[!UICONTROL My Quotes]** não mostra cotações quando algumas cotações contêm produtos excluídos. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4609. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A página **[!UICONTROL My Quotes]** não mostra cotações quando algumas cotações contêm produtos excluídos.

<u>Pré-requisitos</u>:

Os recursos de Cotação B2B e da empresa estão habilitados.

<u>Etapas a serem reproduzidas</u>:

1. Crie uma empresa.
1. Crie vários produtos simples.
1. Crie um produto agrupado com pelo menos duas opções e selecione **[!UICONTROL User Defined]**. Atribua os produtos simples a opções de pacote diferentes.
1. Faça logon na loja como o administrador da empresa.
1. Adicione o produto agrupado ao carrinho duas vezes usando as mesmas opções, mas em quantidades diferentes.
1. Navegue até o carrinho e crie uma cotação.
1. Excluir os produtos simples associados.
1. Para reproduzir o problema, a cotação deve ser sinalizada como `trigger_recollect: 1`.

   Execute o seguinte comando SQL no banco de dados do Adobe Commerce:

   ```
   UPDATE quote SET trigger_recollect = 1 WHERE entity_id = 1;
   ```

1. Vá para **[!UICONTROL My Account]** > **[!UICONTROL My Quotes]**.

<u>Resultados esperados</u>:

A grade **[!UICONTROL My Quotes]** deve ser renderizada com êxito.

<u>Resultados reais</u>:

1. A grade **[!UICONTROL My Quotes]** não foi preenchida.
1. Resposta HTTP (`negotiable_quote/ui/render/`):

   ```
   {"error":"Requested quote is not found. Row ID: quoteId = 1","errorcode":"0"}
   ```

1. Erro no `exception.log`:

   ```
   report.CRITICAL: Magento\Framework\Exception\NoSuchEntityException: Requested quote is not found. Row ID: quoteId = 1 in app/code/Magento/NegotiableQuote/Model/NegotiableQuoteManagement.php:673
   ```
