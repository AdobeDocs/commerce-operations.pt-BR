# ACP2E-4513: Imagens CAPTCHA expiradas não são removidas do sistema

O patch ACP2E-4513 corrige o problema em que imagens CAPTCHA expiradas não são excluídas pelo trabalho cron `captcha_delete_expired_images` e permanecem no sistema. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4513. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

As imagens CAPTCHA expiradas não são automaticamente excluídas pelo trabalho cron `captcha_delete_expired_images` e permanecem no sistema.

<u>Etapas a serem reproduzidas</u>:

1. Vá para **[!UICONTROL Admin]** > **[!UICONTROL Stores]** > **[!UICONTROL Configuration]** > **[!UICONTROL Advanced]** > **[!UICONTROL Admin]** > **[!UICONTROL Captcha]**.
   1. Defina **[!UICONTROL Enable CAPTCHA in Admin]** como *Sim*.
   1. Selecione **[!UICONTROL Forms]** : *Logon de Administrador*, *Senha esquecida de administrador*.
   1. Clique em **[!UICONTROL Save config]**.
1. Saia e abra a página **[!UICONTROL Forgot password]**.
1. Insira credenciais aleatórias e gere algumas imagens captcha.
1. Verifique se as imagens foram criadas no diretório `pub/media/captcha/admin/`.
1. Execute o trabalho cron `captcha_delete_expired_images` e verifique o diretório novamente.

<u>Resultados esperados</u>:

As imagens CAPTCHA expiradas são removidas pelo trabalho cron `captcha_delete_expired_images`.

<u>Resultados reais</u>:

As imagens CAPTCHA expiradas permanecem no diretório após a execução do trabalho cron `captcha_delete_expired_images`.
