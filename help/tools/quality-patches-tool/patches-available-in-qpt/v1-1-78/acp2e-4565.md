# ACP2E-4565: A consulta Company GraphQL retorna &quot;O cliente atual não está autorizado&quot; quando o cabeçalho X-Adobe-Company é usado

O patch ACP2E-4565 corrige o problema em que a consulta do GraphQL da Empresa retorna o erro &quot;O cliente atual não está autorizado&quot; quando ambos os cabeçalhos X-Adobe-Company e Store foram incluídos na solicitação, mesmo que o cliente esteja atribuído corretamente à empresa e à loja especificadas. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4565. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A consulta da GraphQL Empresa retorna &quot;O cliente atual não está autorizado&quot; quando ambos os cabeçalhos X-Adobe-Empresa e Loja são incluídos na solicitação, mesmo que o cliente esteja atribuído corretamente à empresa e loja especificadas.

<u>Etapas a serem reproduzidas</u>:

1. Habilitar empresa B2B.
1. Crie um site adicional, loja e visualização de loja.
1. Crie duas empresas (Empresa um e Empresa dois).
1. Crie um usuário atribuído ao segundo site.
1. Atribuir ambas as empresas ao usuário.
1. Execute a seguinte consulta do GraphQL com esses cabeçalhos:
   * `X-Adobe-Company: MQ==`
   * `Store: storeview2`

   ```graphql
   query Company {
     company {
       id
       legal_name
       name
     }
   }
   ```

<u>Resultados esperados</u>:

Os dados da empresa foram retornados com êxito.

<u>Resultados reais</u>:

A resposta do GraphQL retorna um erro de autorização:

```
{
    "errors": [
        {
            "message": "The current customer isn't authorized.",
            "extensions": {
                "category": "graphql-authorization"
            }
        }
    ]
}
```
