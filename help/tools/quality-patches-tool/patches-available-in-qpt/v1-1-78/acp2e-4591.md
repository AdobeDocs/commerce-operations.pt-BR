# ACP2E-4591: Segmentos do cliente com base na contagem de pedidos, como &quot;Compradores pela primeira vez&quot;, não são atualizados para pedidos feitos por meio da API REST

O patch do ACP2E-4591 corrige o problema em que os segmentos de clientes com base na contagem de pedidos, como &quot;Compradores pela primeira vez&quot;, não são atualizados quando os pedidos são feitos por meio da API REST devido à ausência de uma sessão do cliente. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4591. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p4

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.7-p9

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Os segmentos de clientes com base na contagem de pedidos, como &quot;Compradores pela primeira vez&quot;, não são atualizados quando os pedidos são feitos por meio da API REST.

<u>Etapas a serem reproduzidas</u>:

1. Crie um segmento de cliente chamado *Primeiro Usuário* com a seguinte condição:

   ```
   If ALL of these conditions are TRUE:
   Total Number of Orders less than 1 while ALL of these Conditions match:
   Order Status is Any
   ```

1. Crie um cliente.
1. Execute o cliente `matchCustomerSegmentProcessor` e confirme se ele aparece na grade **[!UICONTROL Matched Registered Customers]**.
1. Faça logon como cliente e adicione um produto ao carrinho pela loja.
1. Faça um pedido usando os endpoints da REST API.
1. Execute o mesmo consumidor novamente.

<u>Resultados esperados</u>:

Os clientes são removidos do segmento *Primeiro Usuário* imediatamente após a realização de um pedido, sem depender do consumidor.

<u>Resultados reais</u>:

O cliente permanece no segmento *Primeiro usuário* mesmo depois de fazer um pedido e executar o consumidor.
