# ACP2E-4732: A indexação parcial é interrompida para clientes com muitas atualizações quando a coluna version_id na tabela changelog atinge seu valor máximo

O patch ACP2E-4732 corrige o problema em que a indexação parcial é interrompida para clientes com um grande número de atualizações quando a coluna version_id na tabela changelog atinge seu valor máximo. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78 está instalado. A ID do patch é ACP2E-4732. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.7-p9

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A indexação parcial para clientes com um grande número de atualizações depois que a coluna `version_id` na tabela `changelog` atinge seu valor máximo.

<u>Pré-requisitos</u>:

1. A instância é executada no modo de produção.
1. Os indexadores estão definidos para o modo *Atualizar na Agenda*.

<u>Etapas a serem reproduzidas</u>:

1. Crie um produto e atualize-o 4.294.967.295 vezes (o valor inteiro máximo sem sinal no MySQL).
1. Atualize o produto mais uma vez.

<u>Resultados esperados</u>:

1. Todas as atualizações são sincronizadas corretamente ao usar o modo *Atualização na Programação*.

<u>Resultados reais</u>:

1. Os produtos não são exibidos na Loja.
1. A tabela `catalog_product_index_price` não contém entradas de preço para esses produtos.
