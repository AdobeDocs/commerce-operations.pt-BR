# ACSD-69541: a quantidade de itens do carrinho não pode ser atualizada via GraphQL após reduzir o estoque de produtos em [!UICONTROL Admin]

O patch ACSD-69541 corrige o problema em que reduzir a quantidade de um produto no painel [!UICONTROL Admin] para menos do que a quantidade já adicionada ao carrinho de compras impede a atualização da quantidade do item do carrinho por meio do GraphQL. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 está instalado. A ID do patch é ACSD-69541. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p2

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Depois que um produto é adicionado ao carrinho, reduzir sua quantidade no painel [!UICONTROL Admin] impossibilita a atualização da quantidade de itens do carrinho por meio do GraphQL.

<u>Etapas a serem reproduzidas</u>:

1. Crie um produto simples: **SKU** = *p1* com **quantidade** = *10*.
1. Reindexe para garantir que **[!UICONTROL salable quantity]** = *10* exista.
1. Adicionar quantidade = *10* do produto criado anteriormente ao carrinho por solicitação do GraphQL.
1. No painel [!UICONTROL Admin], edite o produto e reduza sua quantidade para *5* e, em seguida, salve.
1. Reindexe novamente.
1. Tente atualizar a quantidade do produto no carrinho de compras para *&lt;= 5* usando uma solicitação GraphQL.
1. Tente atualizar a quantidade do produto no carrinho de compras para *> 5* usando uma solicitação GraphQL.

<u>Resultados esperados</u>:

A quantidade de itens do carrinho é atualizada quando a quantidade suficiente é *Em Estoque*. Se a quantidade insuficiente estiver disponível, uma mensagem de erro válida será retornada na resposta da GraphQL.

<u>Resultados reais</u>:

Ao atualizar a quantidade de itens do carrinho para *&lt;= 5*, ocorre uma resposta de *sucesso* do GraphQL, mas a quantidade de produtos no carrinho de compras permanece inalterada.
Ao atualizar a quantidade de itens do carrinho para *> 5*, um erro interno do servidor é retornado em vez de uma mensagem de erro válida.
