# ACSD-69308: as regras de preço do catálogo não se aplicam quando `special_price` é definido no nível do site

O patch ACSD-69308 corrige o problema em que as regras de preço do catálogo não se aplicavam quando `special_price` era definido somente no nível do site (não em &quot;Todas as exibições da loja&quot;). Após a correção, as regras de preço do catálogo são aplicadas corretamente verificando primeiro a loja padrão do site. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 está instalado. A ID do patch é ACSD-69308. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

As regras de preço de catálogo não se aplicavam quando `special_price` era configurado somente no nível do site. O sistema não reconheceu o preço corretamente para os produtos atribuídos a uma exibição de loja específica, resultando em preços inconsistentes.

<u>Etapas a serem reproduzidas</u>:

1. Faça logon no Painel de administração.
1. Crie um site personalizado, uma Exibição de armazenamento e uma Exibição de armazenamento (web2).
1. Definir **[!UICONTROL Catalog Price Scope]** como *Site*.
1. Defina **[!UICONTROL Use for Promo Rule Conditions]** como *Sim* para o atributo `special_price`.
1. Crie a seguinte regra de preço de catálogo.

```
Website: web2
Conditions:
Special Price  greater than  15
Special Price  less than  50
Actions:
Apply: Apply as percentage of original
Discount Amount: 100
```

1. Crie dois produtos atribuídos somente à exibição de loja da **web2**.

Produto 01:

```
Set Advanced Pricing:
All Store View Scope: 30.00
web2 scope: 40.00
```

Produto 02:

```
Set Advanced Pricing:
All Store View Scope: keep empty
web2 scope: 40.00
```

1. Reindexe e carregue cada produto na vitrine da **web2**.

<u>Resultados esperados</u>:

A regra de preço de catálogo deve ser aplicada à exibição da loja da web2. O produto 02 deve mostrar um preço de *0.00*, pois seu preço especial corresponde à condição da regra de catálogo.

<u>Resultados reais</u>:

* Produto 01: O preço é exibido como *0.00*, que corresponde à condição da regra de catálogo.
* Produto 02: O preço é exibido como *40.00*, que é **não esperado**, pois a regra de catálogo deve ser aplicada com base no preço especial definido no nível do site.
