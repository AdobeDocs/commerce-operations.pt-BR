# ACSD-69333: alterações de SKU foram permitidas para produtos com uma Atualização agendada ativa

O patch ACSD-69333 corrige o problema em que as alterações de SKU eram permitidas em produtos com uma Atualização agendada ativa. Após a correção, as alterações da SKU são proibidas durante as atualizações programadas ativas. Ocorreu um erro ao salvar o produto, e o campo SKU do produto está desabilitado para edição no painel [!UICONTROL Admin]. Isso evita inconsistências de Inventário MSI causadas por alterações de SKU durante reversões de preparo. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 está instalado. A ID do patch é ACSD-69333. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p6

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Alterações de SKU foram permitidas para produtos com uma Atualização agendada ativa. Após a correção, as alterações da SKU são proibidas durante as atualizações programadas ativas. Ocorreu um erro ao salvar o produto, e o campo SKU do produto está desabilitado para edição no painel [!UICONTROL Admin]. Isso evita inconsistências de Inventário MSI causadas por alterações de SKU durante reversões de preparo.

<u>Pré-requisitos</u>:

Os módulos de inventário estão instalados e ativados.

<u>Etapas a serem reproduzidas</u>:

1. Faça logon no painel [!UICONTROL Admin].
1. Crie uma nova fonte e um novo estoque.
1. Crie um produto simples. (Por exemplo: SKU = *SP1*)
1. Criar uma atualização de preparo. (Por exemplo: altere o nome.)
1. Enquanto a atualização de preparo estiver ativa, altere o SKU do produto = *SP11*.
1. Verifique a tabela `inventory_source_item`. Outro registro é criado com o novo SKU = *SP11*.
1. Faça um pedido que inclua todas as quantidades disponíveis para tornar o produto **Sem Estoque**.
1. Crie a fatura e a remessa e verifique a tabela `inventory_source_item`.
1. Verifique se o SKU volta para *SP1* e se a disponibilidade de estoque é **No Estoque**.
1. Acesse a Loja e verifique o status do estoque do produto.

<u>Resultados esperados</u>:

O status do estoque do produto não é alterado.

<u>Resultados reais</u>:

A quantidade em estoque é atualizada, e o produto é movido para **Em Estoque**.
