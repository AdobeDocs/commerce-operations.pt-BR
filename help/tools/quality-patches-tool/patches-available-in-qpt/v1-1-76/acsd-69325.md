# ACSD-69325: modificar o caso do SKU fez com que o produto parecesse indisponível na loja

O patch ACSD-69325 corrige o problema em que alterar um SKU do produto para usar caracteres em minúsculas faz com que o produto seja exibido como indisponível na loja. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 está instalado. A ID do patch é ACSD-69325. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Quando o SKU é modificado para incluir letras minúsculas, o produto aparece incorretamente como indisponível na loja.

<u>Etapas a serem reproduzidas</u>:

1. Faça logon no Painel de administração.
1. Crie um site, loja e exibição de loja adicionais navegando até **[!UICONTROL Stores]** > **[!UICONTROL All Stores]**.
1. Crie um canal de vendas para o novo site navegando até **[!UICONTROL Channels]** > **[!UICONTROL Sales Channels]**.
1. Crie um produto atribuído ao novo site com SKU = &#39;RF-42R-80X100-CL&#39; e atribua-o ao canal de vendas.
1. Execute os indexadores `php bin/magento indexer:reindex` e verifique se as tarefas agendadas (cron) estão em execução.
1. Abra a página do produto na loja.
1. Verifique se o produto está *No Estoque*.
1. Atualizar o SKU para `RF-42R-80x100-CL` (usando uma caixa baixa **x**).
1. Execute os indexadores `php bin/magento indexer:reindex` e verifique se as tarefas agendadas (cron) estão em execução.
1. Abra a página do produto na loja.
1. Verifique o status do estoque do produto.

<u>Resultados esperados</u>:

O produto permanece *Em Estoque* na loja.

<u>Resultados reais</u>:

O produto é exibido como *Sem Estoque* na vitrine.
