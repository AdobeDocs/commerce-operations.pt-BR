# ACSD-69086: falha na instalação do MariaDB 10.11 devido à verificação de versão do banco de dados não compatível

O patch ACSD-69086 corrige o problema em que a instalação da instância do Adobe Commerce falha ao usar MariaDB 10.11. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 está instalado. A ID do patch é ACSD-69086. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.7-p8

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A instalação da instância do Adobe Commerce falha ao usar o MariaDB 10.11.

<u>Etapas a serem reproduzidas</u>:

1. Instale o Adobe Commerce no MariaDB versão 10.11.
1. Execute o processo de instalação padrão.
1. Executar `setup:upgrade` após a instalação.

<u>Resultados esperados</u>:

Instalação e `setup:upgrade` concluídas com êxito no MariaDB versão 10.11.

<u>Resultados reais</u>:

1. A instalação no MariaDB 10.11 falha durante a instalação do schema de banco de dados.
1. O processo é interrompido com um erro *Não há suporte para a versão atual do RDBMS*.

## Aplicar o patch

Para aplicar patches individuais, use os links a seguir, dependendo do método de implantação:

* Adobe Commerce ou Magento Open Source local: [[!DNL Quality Patches Tool] > Uso](/help/tools/quality-patches-tool/usage.md) no guia [!DNL Quality Patches Tool].
* Adobe Commerce na infraestrutura em nuvem: [Atualizações e patches > Aplicar patches](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/upgrade/apply-patches.html) no guia do Commerce na infraestrutura em nuvem.
