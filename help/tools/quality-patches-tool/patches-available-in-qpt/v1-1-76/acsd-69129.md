# ACSD-69129: falha na atualização de preço da camada de API [!DNL REST] após excluir o site base padrão e usar o site secundário como padrão

O patch ACSD-69129 corrige o problema em que a exclusão do site base padrão e o uso do site secundário como padrão resultavam em um erro ao tentar atualizar o preço da camada para o site secundário por meio da API [!DNL REST]. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 está instalado. A ID do patch é ACSD-69129. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6-p13, 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.5 - 2.4.6-p13, 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Excluir o site base padrão e usar o site secundário como padrão resulta em um erro ao tentar atualizar o preço da camada para o site secundário por meio da API [!DNL REST].

<u>Etapas a serem reproduzidas</u>:

1. Faça logon no painel [!UICONTROL Admin].
1. Definir **[!UICONTROL Catalog Price Scope]** = *Site*.
1. Crie um novo site/loja/loja.
1. Definir *Definir como padrão* para o site recém-criado.
1. Limpe o cache.
1. Crie um novo produto. Atribua-o somente ao segundo site.
1. Envie uma solicitação de API [!DNL REST] para definir o preço da camada para o segundo site.
1. O resultado deve ser uma matriz vazia com uma resposta *200*.
1. No [!UICONTROL Admin], exclua o *Site Principal*.
1. Envie uma solicitação de API [!DNL REST] para definir o preço da camada para o segundo site.

<u>Resultados esperados</u>:

* Uma resposta *200* foi recebida.
* O preço da camada do produto é atualizado.

<u>Resultados reais</u>:

* O *ID de Site de atributo inválido = %websiteId. ID da Linha: SKU = %SKU, ID do Site: %websiteId, Grupo de Clientes: %customerGroup, Quantidade: %qty.* erro retornado na resposta.
* O preço da camada do produto não é atualizado.
