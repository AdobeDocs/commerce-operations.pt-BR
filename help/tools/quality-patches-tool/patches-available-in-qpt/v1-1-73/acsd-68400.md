# ACSD-68400: a quantidade de cartão-presente virtual não é refletida com precisão na tabela de reserva de estoque.

O patch ACSD-68400 corrige o problema em que a quantidade de cartão-presente virtual não é refletida com precisão na tabela de reserva de estoque. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73 está instalado. A ID do patch é ACSD-68400. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6-p12

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.7-p8

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A quantidade de cartão-presente virtual não está refletida com precisão na tabela de reservas de estoque.

<u>Etapas a serem reproduzidas</u>:

1. Crie uma instância limpa do 2.4-develop.
1. Faça logon no administrador.
1. Crie um cartão-presente virtual.
1. Edite o produto e clique em **[!UICONTROL Advanced Inventory]**.
1. Ative Gerenciar estoque e defina a quantidade como 9999.
1. Salve o produto
1. Vá para o Front-end e navegue até a categoria Cartão-presente.
1. Adicione um produto com uma quantidade de 25.
1. Adicione um produto com uma quantidade de 50.
1. Vá para o checkout e faça o pedido.
1. Verifique a última entrada na tabela `inventory_reservation` para o pedido feito. Ele deve mostrar uma quantidade de 2.
1. Volte para o painel de administração e abra o pedido feito de **[!UICONTROL Sales]** > **[!UICONTROL Orders]**.
1. Clique em **[!UICONTROL Invoice]** e envie uma fatura para o pedido
1. Verifique a tabela `inventory_reservation` novamente. Agora, ele mostra uma reserva de apenas 1 em vez de 2.

<u>Resultados esperados</u>:

A reserva deve ser criada para a quantidade que foi solicitada e faturada, que é 2 nesse caso.

<u>Resultados reais</u>:

A reserva é criada apenas para 1 em vez de 2.
