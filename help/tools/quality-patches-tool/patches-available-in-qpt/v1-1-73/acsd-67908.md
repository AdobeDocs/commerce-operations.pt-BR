# ACSD-67908: os arquivos do JavaScript não são mesclados corretamente em configurações de várias lojas

O patch ACSD-67908 corrige o problema em que os arquivos do JavaScript não são mesclados corretamente em configurações de várias lojas. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73 está instalado. A ID do patch é ACSD-67908. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p5

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Os arquivos do JavaScript mesclados não são gerados corretamente nas configurações de várias lojas, resultando em arquivos corrompidos e erros de vitrine.

<u>Etapas a serem reproduzidas</u>:

1. Configure 50 ou mais sites, lojas e visualizações de loja.
1. Definir modo de produção e ignorar compilação:

   ```
   bin/magento deploy:mode:set production --skip-compilation
   ```

1. Implantar código gerado:

   ```
   bin/magento setup:di:compile
   ```

1. Implante conteúdo estático usando uma estratégia compacta:

   ```
   bin/magento setup:static-content:deploy --strategy compact --jobs 10
   ```

1. Excluir conteúdo da pasta de cache mesclada:

   ```
   rm -rf pub/static/_cache/merged/*
   ```

1. Limpar cache:

   ```
   bin/magento cache:clean
   ```

1. Abra todas as exibições da loja simultaneamente em guias do navegador ou janelas separadas.


<u>Resultados esperados</u>:

Todos os arquivos do JavaScript foram mesclados com êxito, e a loja funciona sem erros.

<u>Resultados reais</u>:

Alguns arquivos JavaScript mesclados estão corrompidos e não começam com a declaração correta do módulo RequireJS.

Para aplicar patches individuais, use os links a seguir, dependendo do método de implantação:
