# ACP2E-4472: Cotação nula criada usando o fluxo &quot;Login como Cliente&quot;

O patch ACP2E-4472 corrige o problema em que registros de aspas nulas são criados na tabela de aspas durante o fluxo **[!UICONTROL Login as Customer]**. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4472. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Um registro de cotação nulo é criado na tabela de cotação durante o fluxo [!UICONTROL Login as Customer].

<u>Etapas a serem reproduzidas</u>:

1. Crie dois clientes.
1. Vá para **[!UICONTROL Admin]** > **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**.
1. Na grade de clientes, selecione um cliente e clique em **[!UICONTROL Login as Customer]**.
1. Depois de ser redirecionado para a loja, verifique os dados na tabela de banco de dados `quote`.
1. Sem fechar a sessão atual da loja, retorne à grade de clientes **[!UICONTROL Admin]**, selecione o outro cliente e clique em **[!UICONTROL Login as Customer]**.

<u>Resultados esperados</u>:

Somente uma cotação associada ao cliente conectado é criada na tabela de banco de dados `quote`.

<u>Resultados reais</u>:

Juntamente com a cotação do cliente, um registro de cotação nula é criado na tabela do banco de dados `quote`.
