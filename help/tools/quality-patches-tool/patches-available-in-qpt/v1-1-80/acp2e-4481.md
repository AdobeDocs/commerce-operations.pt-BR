# ACP2E-4481: Capacidade de venda do produto do pacote recalculada incorretamente após o cancelamento do pedido

O patch ACP2E-4481 corrige o problema em que a venda do produto do pacote é recalculada incorretamente após o cancelamento do pedido. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4481. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.8-p5

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A capacidade de venda do produto do pacote não é recalculada corretamente depois que um pedido é cancelado.

<u>Pré-requisitos</u>:

MSI (Multi-Source Inventory) ativado.

<u>Etapas a serem reproduzidas</u>:

1. Crie um produto simples com uma quantidade em estoque de *6* e um limite de *5* esgotado.
1. Verifique se a quantidade vendável calculada é *1*.
1. Crie um produto agrupado com uma opção necessária que inclua o produto simples com uma quantidade padrão de *1*.
1. Verifique se os produtos simples e de pacote estão visíveis na Loja.
1. Faça um pedido do produto do pacote.
1. Verifique se os produtos simples e de pacote estão esgotados.
1. Cancelar o pedido do Administrador sem criar uma remessa.

<u>Resultados esperados</u>:

O produto simples e o pacote são restaurados ao status em estoque após o cancelamento do pedido.

<u>Resultados reais</u>:

1. Somente o produto simples é restaurado para o status em estoque.
1. O produto do pacote permanece indisponível e sem estoque na Loja.
