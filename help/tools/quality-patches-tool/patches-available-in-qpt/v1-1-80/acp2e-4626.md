# ACP2E-4626: os arquivos do Storefront JavaScript são carregados duas vezes, causando cargas duplicadas e comportamento instável

O patch ACP2E-4626 corrige o problema em que os arquivos JavaScript do storefront são solicitados e executados duas vezes, causando cargas duplicadas intermitentes e comportamento instável.Após a correção, as dependências do JavaScript são carregadas apenas uma vez, eliminando solicitações duplicadas e preservando a funcionalidade de mixin existente. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4626. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.8-p5

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Problema em que alguns arquivos JavaScript do Storefront eram solicitados e executados duas vezes, o que causava cargas duplicadas intermitentes e comportamento instável. Após a correção, as dependências são carregadas apenas uma vez, eliminando solicitações duplicadas e preservando a funcionalidade de mixin existente.&quot;

<u>Pré-requisitos</u>:

1. MSI instalado.

<u>Etapas a serem reproduzidas</u>:

1. Abra qualquer página de vitrine da loja no navegador.
1. Abra o navegador **[!UICONTROL DevTools]** e selecione a guia **[!UICONTROL Console]**.
1. Execute o seguinte comando.

   ```
   require.config({ deps: ['mage/common'] });
   ```

<u>Resultados esperados</u>:

Não ocorrem carregamentos duplicados. A chamada retorna a função necessária e nenhuma nova solicitação de script é gerada.

<u>Resultados reais</u>:

Novas solicitações aparecem na guia **[!UICONTROL Network]** (por exemplo, mage/common.js, requirejs/domReady.js, jquery.js).
