# ACP2E-4488: Salvar/editar o produto Administração Lenta para conjuntos de atributos grandes

O patch do ACP2E-4488 corrige o problema em que salvar ou editar produtos no Admin é lento para produtos com conjuntos de atributos grandes. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4488. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Salvar ou editar produtos com conjuntos de atributos grandes é um processo lento no Administrador.

<u>Etapas a serem reproduzidas</u>:

1. Crie pelo menos 500 atributos de produto e atribua-os a um conjunto de atributos.
1. Vá para o **Painel de administração**.
1. Crie um novo produto ou edite um produto existente usando o conjunto de atributos com muitos atributos.
1. Clique em **Salvar**.

<u>Resultados esperados</u>:

1. A operação de salvar ou editar o produto é concluída com sucesso sem erros do navegador.
1. A página de Administrador permanece responsiva durante o processo de salvamento.

<u>Resultados reais</u>:

1. A página de Administrador não responde ao salvar ou editar o produto.
1. Um aviso de navegador *Página sem resposta* é exibido.

## Aplicar o patch

Para aplicar patches individuais, use os links a seguir, dependendo do método de implantação:

* Adobe Commerce ou Magento Open Source local: [[!DNL Quality Patches Tool] > Uso](/help/tools/quality-patches-tool/usage.md) no guia [!DNL Quality Patches Tool].
* Adobe Commerce na infraestrutura em nuvem: [Atualizações e patches > Aplicar patches](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/upgrade/apply-patches.html) no guia do Commerce na infraestrutura em nuvem.
