# ACP2E-4552: A resposta do GraphQL não retorna o status da empresa

O patch ACP2E-4552 corrige o problema em que o status da empresa não é retornado na resposta do GraphQL. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4552. Observe que este problema está programado para ser corrigido no Adobe Commerce B2B 1.5.2-p3.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) B2B 1.5.2-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) B2B >=1.5.0 &lt;1.5.3

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A resposta da GraphQL não retorna o status da empresa.

<u>Pré-requisitos</u>:

[!DNL B2B] está instalado.

<u>Etapas a serem reproduzidas</u>:

1. No Administrador, vá para **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]** e defina **[!UICONTROL Enable Company]** como `Yes`.
1. Faça logon no Commerce Admin como administrador.
1. Crie três empresas com esses status: Ativa, Rejeitada e Aprovação pendente.
1. No Admin, vá para **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**. Abra uma conta de cliente e, na lista **[!UICONTROL Associate to Company]**, atribua o cliente a cada uma das empresas criadas.
1. Envie a seguinte solicitação do GraphQL:

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies {
        items {
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>Resultados esperados</u>:

A resposta do GraphQL retorna os valores de status corretos (como Ativo, Aprovação pendente, Rejeitado e Bloqueado) para todas as empresas associadas ao cliente. A resposta também pode ser filtrada ou classificada por status da empresa.

<u>Resultados reais</u>:

A resposta da GraphQL não inclui os valores de status das empresas associadas ao cliente.
