# ACP2E-4496: O trabalho CRON do Analytics degrada o desempenho durante a execução

O patch ACP2E-4496 corrige o problema em que o trabalho cron do Analytics causa degradação de desempenho durante a execução, resultando em um melhor desempenho geral do sistema. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4496. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

O trabalho cron do Analytics causa degradação do desempenho durante a execução e aumenta a carga do banco de dados.

<u>Etapas a serem reproduzidas</u>:

1. Habilite a análise MBI.
1. Gerar uma grande quantidade de entradas na tabela `sales_order_item`.
1. Execute o cron `analytics_collect_data` e meça o tempo necessário para a conclusão.

<u>Resultados esperados</u>:

O cron de análise deve levar um tempo razoável para ser concluído.

<u>Resultados reais</u>:

O cron do Analytics leva muito tempo para ser concluído e coloca uma carga alta no banco de dados.
