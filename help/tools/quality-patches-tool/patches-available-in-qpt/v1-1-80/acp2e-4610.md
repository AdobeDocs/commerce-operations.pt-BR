# ACP2E-4610: Execução lenta do cron `sales_clean_quotes`

O patch ACP2E-4610 corrige o problema em que o trabalho cron `sales_clean_quotes` tem problemas de desempenho. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4610. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6-p13

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

O trabalho cron `sales_clean_quotes` tem problemas de desempenho.

<u>Etapas a serem reproduzidas</u>:

1. Gerar várias cotações expiradas na tabela de cotações.
1. Execute o trabalho cron `sales_clean_quotes` e meça o tempo de execução.

<u>Resultados esperados</u>:

O trabalho cron `sales_clean_quotes` termina dentro de um tempo razoável.

<u>Resultados reais</u>:

O trabalho do cron `sales_clean_quotes` demora muito para ser concluído.

Para aplicar patches individuais, use os links a seguir, dependendo do método de implantação:
