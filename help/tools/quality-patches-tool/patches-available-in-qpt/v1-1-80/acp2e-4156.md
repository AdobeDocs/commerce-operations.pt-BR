# ACP2E-4156: a validação do endereço de entrega da API [!DNL REST] ignora a configuração do atributo Admin

O patch ACP2E-4156 corrige o problema em que a validação do endereço de remessa na API [!DNL REST] não adere à configuração de atributo definida em Administração. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80 está instalado. A ID do patch é ACP2E-4156. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p4

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Na API [!DNL REST], a validação do endereço de entrega não respeita as configurações do atributo de endereço do cliente definidas no Administrador.

<u>Etapas a serem reproduzidas</u>:

1. Vá para **[!UICONTROL Stores]** > **[!UICONTROL Attributes]** > **[!UICONTROL Customer Address]** no painel Admin.
1. Abra o atributo com o código de atributo *firstname*.
1. Em **[!UICONTROL Attribute Properties]**, defina:
   1. **[!UICONTROL Input Validation]**: Somente Comprimento.
   1. **[!UICONTROL Minimum Text Length]**: *1*.
   1. **[!UICONTROL Maximum Text Length]**: *20*.
1. Acesse a loja e adicione o produto de teste ao carrinho.
1. Envie uma solicitação de API [!DNL REST] a `/rest/default/V1/guest-carts/cartId/shipping-information` para adicionar ou atualizar informações de envio com um valor de *firstname* com mais de 20 caracteres.

<u>Resultados esperados</u>:

Um erro é retornado indicando que o valor *firstname* excede o limite de comprimento configurado.

<u>Resultados reais</u>:

As informações de envio do carrinho de compras foram atualizadas com êxito com o valor *firstname* enviado na solicitação de API [!DNL REST].
