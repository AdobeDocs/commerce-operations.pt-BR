# ACSD-68430: Falha ao salvar um cliente ou endereço de cliente se o registro incluísse várias opções de atributo que tinham sido excluídas da configuração do atributo

O patch ACSD-68430 corrige um problema em que falha ao salvar um cliente ou endereço de cliente se o registro incluir várias opções de atributo que foram excluídas da configuração de atributo. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74 está instalado. A ID do patch é ACSD-68430. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p6

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.6 - 2.4.7-p8

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Ao salvar um cliente ou endereço de cliente que inclui várias opções de atributo que foram excluídas da configuração de atributo, a operação falha.

<u>Etapas a serem reproduzidas</u>:

1. Navegue até Admin e crie um atributo de cliente personalizado de multisseleção (exemplo: &quot;teste&quot;) com as opções A, B, C, D.
1. Crie um cliente em Admin e salve os valores para esse atributo (exemplo: selecione A, B, C).
1. Retorne à configuração do atributo e exclua uma das opções que foram salvas no cliente (exemplo: excluir opção C).
1. Em Admin, tente editar o mesmo registro do cliente - altere qualquer campo (como gênero ou prefixo) e clique em **[!UICONTROL Save]**.

<u>Resultados esperados</u>:

As alterações no registro do cliente são salvas sem erros.

<u>Resultados reais</u>:

Erro ao salvar alterações no registro do cliente: *O atributo [nome_do_atributo] não contém a opção com a ID [id_da_opção_excluída]*.
