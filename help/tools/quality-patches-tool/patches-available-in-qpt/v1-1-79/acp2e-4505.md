# ACP2E-4505: Dependência de categoria circular ao salvar uma categoria com dados obsoletos em uma guia do navegador duplicada

O patch ACP2E-4505 corrige o problema em que era possível salvar uma categoria com dados obsoletos de uma guia do navegador duplicada, criando uma dependência circular. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 está instalado. A ID do patch é ACP2E-4505. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3, 2.4.7-p8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Era possível salvar uma categoria com dados obsoletos de uma guia do navegador duplicada, criando uma dependência circular.

<u>Etapas a serem reproduzidas</u>:

1. Crie uma estrutura de categoria: A > B > C.
1. No painel Admin, vá para **[!UICONTROL Catalog]** > **[!UICONTROL Categories]**.
1. Na árvore de categorias, selecione a categoria C para abrir seu formulário de edição na guia 1.
1. Duplique a guia do navegador para abrir a guia 2 com o mesmo formulário de edição da categoria C.
1. Na Guia 1:
   1. Arraste a categoria C em A para que C se torne filho de A.
   1. Arraste a categoria B sob C para que B se torne filho de C.
   1. Aguarde até que as operações de arrastar e soltar terminem e a árvore de categorias seja atualizada.
1. Na guia 2 (ainda mostrando a hierarquia antiga):
   1. Faça qualquer alteração, por exemplo, edite a descrição.
   1. Clique em **[!UICONTROL Save]**.

<u>Resultados esperados</u>:

O sistema deve validar e impedir que relações circulares pai-filho sejam salvas.

<u>Resultados reais</u>:

Uma dependência circular pai-filho é criada entre as categorias (a categoria B tem o pai C e a categoria C tem o pai B), causando recursão infinita no percurso da hierarquia da categoria.
