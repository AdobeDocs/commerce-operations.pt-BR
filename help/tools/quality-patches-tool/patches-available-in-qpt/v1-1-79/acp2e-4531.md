# ACP2E-4531: A hierarquia CMS não atualiza o URL hierárquico quando a chave do URL da página secundária é alterada

O patch ACP2E-4531 corrige o problema em que alterar a chave de URL de uma página do CMS não atualizava o URL hierárquico da página. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 está instalado. A ID do patch é ACP2E-4531. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Alterar a chave de URL de uma página do CMS não atualiza o URL hierárquico da página.

<u>Etapas a serem reproduzidas</u>:

1. Crie uma página do CMS (por exemplo, *ajuda*).
1. Crie uma segunda página do CMS (por exemplo, *my-account*) e defina *help* como sua página pai na hierarquia.
1. Carregue ambas as páginas da loja e ambas as páginas serão resolvidas corretamente.
1. Altere a Chave de URL da página minha conta (em *Otimização do Mecanismo de Pesquisa*) para *minha conta-2*.
1. Carregar `/my-account-2` e `/help/my-account-2` da Loja.

<u>Resultados esperados</u>:

Quando a Chave de URL da página do CMS é alterada, os URLs de hierarquia são atualizados de acordo. `/help/my-account-2` resolve para a página, e `/my-account` não resolve mais (a URL não existe).

<u>Resultados reais</u>:

`/my-account-2` redireciona para `/my-account` e `/help/my-account-2` retorna um erro 404. O URL hierárquico não é atualizado quando a chave do URL da página secundária é alterada.
