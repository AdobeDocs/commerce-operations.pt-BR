# ACP2E-4601: Salvando novamente uma autorreferência de transação de autorização de pagamento `parent_id` e acionando um loop/OOM infinito

O patch ACP2E-4601 corrige o problema em que o processamento de transações de pagamento se comporta de forma ineficiente ao salvar novamente as transações de autorização sem um `parent_txn_id`. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79 está instalado. A ID do patch é ACP2E-4601. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8-p3

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

O processamento de transações de pagamento se comporta de forma ineficiente ao salvar novamente as transações de autorização criadas sem um `parent_txn_id`.

<u>Etapas a serem reproduzidas</u>:

1. Configurar um método de pagamento online.
1. Colocar um pedido que crie uma transação de pagamento de autorização sem um `parent_txn_id`.
1. Salve novamente a transação de autorização.

<u>Resultados esperados</u>:

O processamento da transação de pagamento é concluído com êxito.

<u>Resultados reais</u>:

Falha ao salvar novamente a transação de autorização.
