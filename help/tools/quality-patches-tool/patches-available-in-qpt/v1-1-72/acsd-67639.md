# ACSD-67639: Falha ao criar um memorando de crédito para produtos agrupados com **[!UICONTROL Dynamic Price]** definido como *Não*.

O patch ACSD-67639 corrige o problema em que os memorandos de crédito não podem ser criados para produtos de pacote com **[!UICONTROL Dynamic Price]** definido como *Não*. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.72 está instalado. A ID do patch é ACSD-67639. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.8 - 2.4.8-p2

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Falha na criação do memorando de crédito para produtos agrupados quando **[!UICONTROL Dynamic Price]** está definido como *Não*.

<u>Etapas a serem reproduzidas</u>:

1. Criar um produto de pacote com **[!UICONTROL Dynamic Price]** definido como *Não*.
1. Adicione duas opções, cada uma contendo um produto simples.
1. No front-end, adicione o produto do pacote recém-criado ao carrinho.
1. Vá para o check-out e faça o pedido usando **[!UICONTROL Check / Money order]**&quot;.
1. Vá para o administrador e abra o pedido recém-criado.
1. Vá para fatura e clique em enviar fatura.
1. Vá para o memorando de crédito e clique em **[!UICONTROL Refund Offline]**.

<u>Resultados esperados</u>:

O memorando de crédito é criado sem erros.

<u>Resultados reais</u>:

* O memorando de crédito não foi criado.
   * Erro exibido: `We found an invalid quantity to refund item 'bundle_product_name`.
