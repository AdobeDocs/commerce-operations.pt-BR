# ACSD-67383: erro ao fazer logon como um cliente com duas contas de administrador de empresa na mesma sessão

O patch ACSD-67383 corrige o problema em que tentar usar [!UICONTROL Login as Customer] para duas contas de administrador de empresa na mesma sessão sem terminar a primeira sessão resulta em um erro *Nenhuma entidade com cartId*. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.72 está instalado. A ID do patch é ACSD-67383.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p4

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.7-p7

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

Tentar usar **[!UICONTROL Login as Customer]** para duas contas de administrador de empresa sem fechar a primeira sessão resulta em uma exceção *Nenhuma entidade com cartId*. Isso também cria uma entrada isolada na tabela de cotações.

<u>Etapas a serem reproduzidas</u>:

1. Instale uma instância limpa com [!DNL B2B].
1. Crie uma nova Empresa.
1. Crie dois administradores de Empresa separados.
1. Navegue até [!UICONTROL Customers] > [!UICONTROL All Customers].
1. Habilite [!UICONTROL Allow Remote Shopping Assistance] para cada um dos usuários administradores de empresas recém-criados.
1. Selecione um administrador de empresa e clique em [!UICONTROL Login as Customer] para iniciar a sessão de logon.
1. Sem fechar a sessão atual, clique em [!UICONTROL Login as Customer] para que o outro administrador de empresa tente fazer outro logon.

<u>Resultados esperados</u>:

O logon deve ser permitido sem criar registros vazios na tabela de cotações.

<u>Resultados reais</u>:

Uma mensagem de erro, *Nenhuma entidade com cartId = 22,* é retornada e registros vazios são criados na tabela de cotações.
