# ACSD-67424: o valor `updated_at` na resposta da API REST `GET /carts/search` não corresponde ao valor em [!UICONTROL Admin Panel] para [!UICONTROL Negotiable Quotes]

O patch ACSD-67424 corrige o problema em que o valor `updated_at` na resposta da API REST `GET /carts/search` não corresponde ao valor em [!UICONTROL Admin Panel] ao usar **[!UICONTROL Negotiable Quotes]**. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.72 está instalado. A ID do patch é ACSD-67424. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p4

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p3 - 2.4.8-p2

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=pt-BR). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

O valor `updated_at` na resposta da API REST `GET /carts/search` não corresponde ao valor mostrado em [!UICONTROL Admin Panel] ao usar [!UICONTROL Negotiable Quotes].

<u>Etapas a serem reproduzidas</u>:

1. Habilitar *empresa* e *cotação B2B*.
1. Crie uma nova cotação no [!UICONTROL Admin Panel].
1. Adicione qualquer produto à cotação e clique em **[!UICONTROL Send]**.
1. Faça logon no front-end como um usuário da empresa e vá para **[!UICONTROL My Quotes]**.
1. Clique em **[!UICONTROL View action]**, adicione um endereço de entrega e clique em **[!UICONTROL Send for Review]**.
1. No [!UICONTROL Admin Panel], selecione um método de envio para a cotação e clique em **[!UICONTROL Send]**.
1. Volte para **[!UICONTROL My Quotes]** na loja, selecione o **[!UICONTROL Quote]** e clique em **[!UICONTROL Proceed to Checkout]**.
1. Coloque o pedido.
1. Use o valor `updated_at` da tabela de cotações no banco de dados para recuperar informações de cotações por meio da API REST.

   ```
   /rest/V1/carts/search?searchCriteria[filterGroups][0][filters][0][field]=updated_at&searchCriteria[filterGroups][0][filters][0][value]={QUOTE_UPDATED_AT}&searchCriteria[filterGroups][0][filters][0][condition_type]=gteq
   ```

1. Compare o valor `updated_at` da resposta da API com o valor **[!UICONTROL Updated at]** na grade [!UICONTROL Quotes] de [!UICONTROL Admin Panel].

<u>Resultados esperados</u>:

Os valores `updated_at` na resposta da API REST e [!UICONTROL Admin Panel] são os mesmos (considerando as diferenças de fuso horário).

<u>Resultados reais</u>:

A resposta da API REST retorna um valor `updated_at` incorreto que não corresponde ao valor **[!UICONTROL Updated at]** na grade [!UICONTROL Quotes] de [!UICONTROL Admin Panel].
