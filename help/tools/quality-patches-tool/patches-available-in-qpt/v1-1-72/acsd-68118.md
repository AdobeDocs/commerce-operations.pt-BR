# ACSD-68118: `customerCart` [!DNL GraphQL] a consulta retorna valores de atributo de produto incorretos para exibição de loja

O patch ACSD-68118 corrige o problema em que a consulta `customerCart` [!DNL GraphQL] retorna valores de atributo de produto que não refletem a exibição de loja definida no cabeçalho da solicitação **[!UICONTROL Store]**, causando localização inconsistente. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.72 está instalado. A ID do patch é ACSD-68118. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p2

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.4 - 2.4.8-p2

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A consulta [!DNL GraphQL] `customerCart` retorna atributos de produto que ignoram o cabeçalho **[!UICONTROL Store]**, levando à localização incorreta para exibições de loja.

<u>Etapas a serem reproduzidas</u>:

1. Criar uma nova exibição de loja em uma loja existente.
1. Crie um produto.
1. Defina um valor [!UICONTROL product name] diferente para a exibição de armazenamento padrão.
1. Crie uma conta de cliente.
1. Adicione o produto ao carrinho como usuário conectado usando o [!DNL GraphQL].
1. Busque os dados do carrinho para ambas as exibições de loja alterando o cabeçalho **[!UICONTROL Store]**.

<u>Resultados esperados</u>:

Os dados do produto (por exemplo, nome, descrição) devem corresponder ao modo de exibição de loja especificado no cabeçalho **[!UICONTROL Store]**.

<u>Resultados reais</u>:

1. Os dados do produto não refletem os valores específicos da loja.
1. A alteração do cabeçalho **[!UICONTROL Store]** não afeta os atributos do produto retornados.
