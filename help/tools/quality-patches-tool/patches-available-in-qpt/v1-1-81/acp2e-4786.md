# ACP2E-4786: Falha na exportação do produto quando o armazenamento remoto do AWS S3 é configurado

O patch ACP2E-4786 corrige o problema em que a exportação do produto falha porque o driver de armazenamento usa um caminho inválido quando o armazenamento remoto do AWS S3 é ativado. Este patch está disponível quando o [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81 está instalado. A ID do patch é ACP2E-4786. Observe que esse problema está programado para ser corrigido no Adobe Commerce 2.4.9.

## Produtos e versões afetados

**O patch foi criado para a versão do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.7-p4

**Compatível com as versões do Adobe Commerce:**

* Adobe Commerce (todos os métodos de implantação) 2.4.5 - 2.4.9

>[!NOTE]
>
>O patch pode se tornar aplicável a outras versões com as novas versões do [!DNL Quality Patches Tool]. Para verificar se o patch é compatível com a sua versão do Adobe Commerce, atualize o pacote `magento/quality-patches` para a versão mais recente e verifique a compatibilidade na [[!DNL Quality Patches Tool]: página Procurar patches](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Use a ID do patch como palavra-chave de pesquisa para localizar o patch.

## Problema

A exportação do produto falha quando o armazenamento remoto do AWS S3 é habilitado devido à manipulação de caminho inválido no driver de armazenamento.

<u>Etapas a serem reproduzidas</u>:

1. Configurar [!DNL AWS S3] como armazenamento remoto:

   ```
   bin/magento setup:config:set --remote-storage-driver="aws-s3" --remote-storage-bucket="..." --remote-storage-region="..." --remote-storage-key="..." --remote-storage-secret="..." -n
   ```

1. Desative o armazenamento do banco de dados para arquivos de mídia executando o seguinte comando:

   ```
   bin/magento config:set system/media_storage_configuration/media_database 0
   ```

1. Na barra lateral Admin, vá para **[!UICONTROL System]** > **[!UICONTROL Data Transfer]** > **Exportar**.
1. Para **[!UICONTROL Entity Type]**, escolha **[!UICONTROL Products]**.
1. Clique em **[!UICONTROL Continue]**.

<u>Resultados esperados</u>:

O arquivo de exportação foi criado com sucesso.

<u>Resultados reais</u>:

1. O arquivo de exportação não foi criado.
1. Os logs mostram um erro *SignatureDoesNotMatch*.
